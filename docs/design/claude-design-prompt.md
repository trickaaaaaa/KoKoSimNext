# Claude Design 用プロンプト：高校野球監督シミュレーション UIモック

以下をそのままClaude Designに貼り付けてください。3画面（メイン／選手ステータス／データ閲覧）を想定しています。
一度に3画面が重い場合は、「## 画面1」「## 画面2」「## 画面3」を分割して渡してください。

---

高校野球の監督シミュレーションゲームのUIモックを作ってください。プレイヤーは高校野球部の監督として、選手を育成し、采配し、甲子園を目指します。現代的なスポーツマネジメントゲーム（Football Manager系）の情報設計で、データが豊富に詰まっていて、グラフや色分けで一目で状況が掴める画面にしてください。PC横画面（16:9）が主対象です。

## ビジュアルテーマ（厳守）
- コンセプトは「球場のスコアボード」。ベースは電光掲示板の深緑（#14231B 背景、#1D3227 パネル面）、テキストはチョーク白（#EFF4EA）、補助テキストはくすんだ緑灰（#9DB3A0）、枠線は #38543F。
- アクセントは電光ランプのアンバー1色（#F5C64A）。主要な操作・重要数値・警告のハイライトに集中して使う。警告色は #E86A4A。
- 数字やスコアはコンデンス系のスポーツ書体（Oswald等の細長い数字）、見出しは明朝で緊張感、本文はゴシック。数字の存在感を強めに。
- 能力ランクは S〜G の7段階で、固定カラーチップ＋文字併記（色覚対応）。目安：S=マゼンタ寄り、A=赤橙、B=橙、C=アンバー、D=黄緑、E=緑、F=青灰、G=灰。この配色を選手表・チーム評価すべてで共通の視覚言語として使う。
- 過度な角丸やドロップシャドウは避け、電光掲示板らしい端正さ。区切り線と余白で情報を整理する。

## ゲーム世界の前提（架空データで作成。実在の学校・人物は使わない）
- 選手は高校生（1〜3年）。能力は内部1〜100だが表示は S〜G ランク＋一部は実数（球速はkm/h直接表示）。
- 野手の能力：ミート／パワー／弾道／選球眼／走力／バント／盗塁／走塁判断／肩／送球精度／守備／捕球。
- 投手の能力：球速（例 最速141km/h）／コントロール／スタミナ／球種ごとのランク／投手センス。投法はオーバー/サイド/アンダー。
- **重要（能力項目を勝手に増やさないこと）**：上に挙げた能力**以外**は作らないこと。特に以下は能力バー/能力値として作らない——「対左打者」「対右打者」「対ピンチ」「チャンス」などの状況系能力（これらは別システムで内部処理されるため画面に能力バーとして出さない）、「フィールディング/投手の守備」（守備は野手能力の『守備』に一本化）、「回復力」（スタミナで表現）。投手の守備能力は野手能力の『守備』欄で見せればよい。
- 球種はストレート含めG〜Sランク。変化球は複数持てる（スライダー、カーブ、フォーク等）。
- 各選手に「調子」（絶不調〜絶好調の5段階、パワプロ風の↑↑〜↓↓アイコンで表示）、投打（右投左打など）、背番号（1〜20、1=エース/2=正捕手が慣例）、ポジション、学年。
- 特殊能力（スキル）は有無フラグ。例：尻上がり、広角打法、ムードメーカー、故障しにくい、精神的支柱、荒れ球 等。
- 成績は3つの時間軸：今大会成績／高校3年間の公式戦通算／キャリア通算。
- 現代的な指標も出す：野手はOPS・出塁率・長打率・平均打球初速/角度、投手はK/9・WHIP・防御率・平均球速の推移。

## 画面1：メインダッシュボード（週次ホーム）
監督が毎週ここから操作する司令塔画面。上部にスコアボード風ヘッダー（自校名＋都道府県、**自チームの総合ランク（例：Bランク）**、現在の年・週、夏予選までの残り週、資金・名声ランク・信頼度ランク、大きな「今週を進める▶」ボタン）。
※「硬式野球部」等の当たり前の語は表記しない（自校名だけでよい）。
主要ブロックを配置：
- 次の試合カード（対戦カード、日時・球場・天気。※勝率予想は入れない）
- 部の状態サマリ（チーム士気、平均疲労のゲージ、怪我人数。疲労が高い選手は警告色＋点滅ドット）
- 通知フィード（縦スクロールのlog：「三宅の球速が141km/hに上昇」「小野寺の疲労が72、要休養」「早瀬の性格が判明」「地方大会新聞：海鳴工業のエースが好投」等。成長は上向き色、警告は警告色で左端をカラーバー分け）
- 今週の練習計画（月〜日の7日間、各日に練習メニュー、試合日はハイライト）＋個別指導3枠の指名
- 注目選手の小さな一覧（数名、ランクチップと調子アイコン付き）
情報量は多めに、でもカードで区切って整理。ナビゲーションバー（ホーム／選手／練習／試合／大会・全国／スカウト／経営／キャリア／記録）も表示。

## 画面2：選手ステータス（選手詳細）
一人の選手を深掘りする画面。この画面だけで「この選手はどんな奴か」が全部わかるように。
- ヘッダー：背番号（大きく）、名前、学年、ポジション、投打（右投左打等）、現在の調子アイコン、総合ランク。顔グラフィックは使わず、背番号＋ポジション色の記号アイコンで表現。
- **調子アイコンは丸いデフォルメの表情顔（パワプロ風）**：絶好調に近いほど満面の笑顔、普通は無表情、絶不調に近いほどげんなりした顔。5段階の表情で一目で調子が伝わるようにする。
- **能力表示は「ランクチップ＋実数値＋横長バー」の3点セット**：各能力（ミート、パワー…／球速、コントロール…）に S〜G のカラーチップ、その隣に内部数値（例：68）、さらに横長のバーで長さ表現。数値とバーは同じ色系統でランクに連動。
- **野手能力と投手能力を1画面に両方表示**（タブ切替にせず、同一画面に並べる）。二刀流や投手の打撃も一目で見えるように。投手でない選手は投手欄が低い値で表示される（＝運動能力ベースラインがあるので極端にゼロにはならない）。
- **守備位置適性（全9ポジション）**：投手・捕手・一塁・二塁・三塁・遊撃・左翼・中堅・右翼それぞれの適性をランク（S〜G）で表示する。選手のポジションは監督が決める（コンバート可能）ため、9ポジション全ての適性が一目で見える必要がある。見せ方は、ダイヤモンド型の守備配置図に各ポジションをランク色で塗る、または9マスのコンパクトなグリッドにランクチップを並べる形が望ましい（能力バーは使わず省スペースに）。本職ポジションが最も高く、隣接ポジション（内野内・外野内）はそこそこ、内⇔外や投手⇔捕手は低め、という濃淡が自然に出る想定。
- 能力バランスのレーダーチャート（六角形など）も併記。
- 成長推移グラフ：入学時から現在までの主要能力（例：球速、ミート）の折れ線。伸びてきている様子が見える。
- 投手なら **球種チャート（プロスピ/パワプロ風）を投手能力のブロック内に組み込む**：ボールを中心に、各球種を変化する方向へ放射状に配置する（ストレートは上、スライダーは左、フォークは下、シュートは右…など変化方向に対応）。各球種に球種名＋キレのランク（G〜S、カラーチップ）を表示。**球速は表示しない**（球種の質はランクで表現するため）。変化の大きい球種ほど中心から離す等、変化量が視覚的に伝わるとよい。ストレートも1球種としてランク付きで中心上部に置く。※野手（投手でない選手）の場合は球種チャートはストレート中心で、変化球は0〜1個だけ（何球種も持たせない）。
  - ※本職が野手の選手（二刀流・緊急登板要員）の場合は、球種チャートは**ストレート中心＋変化球0〜1個**にとどめる（野手は球速・制球は相応に出るが、変化球のレパートリーは持たないのが自然）。
- スキル欄：保有する特殊能力をタグ＋短い説明（「どんな時に効くか」）。
- 隠しパラメータの推定：性格・成長タイプ・伸びしろが「？→判明」で段階的に開示される様子（一部は？のまま）。
- 簡易成績：今大会／公式戦通算の主要スタッツを小さく。
現代的で、グラフとチップとバーを多用して視覚的に。

## 画面3：データ閲覧画面（成績・ランキング）
「選手成績を眺めてニヤニヤする」ための情報過多を許容した画面。
- タブで時間軸切替：今大会／公式戦通算／キャリア通算。
- ソート・フィルタ可能な選手成績テーブル（打者：打率・本塁打・打点・OPS・盗塁など／投手：防御率・**投球回(イニング)**・奪三振・K/9・WHIP・最速）。※高校野球なので勝敗（勝-敗）は表示しない。投球回を主要指標として載せる（連投・酷使の度合いが見える）。ランクチップと調子アイコンを各行に。
- チーム内ランキングやリーグ内順位のミニチャート（棒グラフ・比較）。
- キャリアハイや記録更新のハイライト表示（「校歴代本塁打記録に迫る」等をアンバーで祝う）。
- 打球初速や球速分布などのヒストグラム/散布図を1つ入れて、データの深さを見せる。
- 卒業生（教え子）がプロで活躍している様子を追える小セクションがあってもよい。
表・グラフ・チップを詰め込みつつ、色分けとレイアウトで見やすさを保つ。
