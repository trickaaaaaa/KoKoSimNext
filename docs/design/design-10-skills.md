# 設計書10: 特殊能力（スキル）詳細設計

選手の"顔"を作る質的な個性。設計書02 §4.5のスキル概念を体系化する。

## 0. 設計原則（最重要・切り分けの鉄則）

- **有無のみ（ランクなし）**: スキルは「持つ／持たない」のフラグ。強弱はパラメータ（連続値）が担う
- **後天取得あり**: 生成時保有に加え、練習・イベントで習得できる。まれに克服も
- **パラメータ／精神力システムで表現できるものはスキルにしない（鉄則）**:
  - 走塁の巧拙 → バント・盗塁・走塁判断パラメータ（設計書01）
  - 送球の正確さ・肩 → 送球精度・肩パラメータ
  - **左右の相性 → 打者の「対左投手経験（慣れ）」＋左投手の希少性（設計書02 §2.3）で既に表現**。スキル化しない
  - **チャンス／逆境／大舞台／満塁／対ピンチ等の状況補正 → 精神力×プレッシャー指数（設計書02 §3）で既に算出**。スキル化しない
  - 制球の安定・被本塁打の傾向 → コントロール・球種の質パラメータ
- スキルに残すのは、上記のどれでも出せない**質的な癖・非線形な挙動・チームへの波及**だけ
- 効果は控えめに。多数の弱スキルの集合体にしない（元祖の硬派さ）

## 1. スキルとして「残るもの」の基準

「これはパラメータや精神力システムで出せるか？」を問い、出せないものだけを残す。

| 残す理由 | 説明 |
|---|---|
| 非線形な挙動 | 数値の線形補正では表せない「効き方の癖」（尻上がり、ムラっけ等） |
| 質的なスタイル | 数値の高低ではない打ち方・投げ方の個性（広角打法、初球から振る等） |
| チームへの波及 | 本人の能力ではなく他選手・チームに作用（ムードメーカー等） |
| 体質・成長 | 能力値ではない体の性質（故障しにくい、練習熱心等） |
| 特定条件の技 | パラメータ化しない稀な状況技（特定投法の職人技等） |

## 2. カテゴリ別ラインナップ（純化後）

### 2.1 打撃系（挙動の癖・スタイルのみ）
- 尻上がり（打席を重ねるほど当たりが出る：試合内の非線形な調子）
- ムラっけ（好不調の振れ幅が大きい：両刃）
- 広角打法（打球方向の分布が広い＝守備シフトを無効化しやすい。方向の質であり打力ではない）
- 初球から振る（初球スイング傾向。積極性という行動特性）
- 粘り打ち（ファウルで粘り球数を稼ぐ：行動特性）
- ※「チャンスに強い」等は精神力で表現するためスキルにしない

### 2.2 投手系（挙動の癖のみ）
- 尻上がり（イニングを追うごとに安定：試合内の非線形）
- 打者一巡（2巡目以降に崩れやすい／逆に対応する：対戦回数依存の癖）
- 荒れ球（軌道が不安定で打ちにくいが四球も増える：両刃の質）
- クセ球（見た目より打ちにくい球質：物理では出しにくい質的個性）
- ※「制球が良い」「打たれ強い」は各パラメータで表現するためスキルにしない

### 2.3 守備系（質的な癖のみ）
- 併殺の名手（二塁経由の送球が速い：連携動作の質）
- 名捕手（配球AIの質を上げる：リードという非数値の技）
- ※「守備範囲が広い」「肩が強い」「送球が正確」は全てパラメータで表現するためスキルにしない

### 2.4 チーム系（本人以外へ波及＝パラメータで出せない）
- ムードメーカー（チーム士気の底上げ）
- 精神的支柱（主将適性。設計書09 §8の統率力に寄与）
- 練習リーダー（周囲の練習効率に微ボーナス）
- お手本（同ポジション後輩の成長を促進）

### 2.5 体質・成長系（能力値でない体の性質）
- 練習熱心（経験値効率アップ）／ サボり癖（負）
- 故障しにくい（怪我判定を軽減。設計書03 §3.5）／ ケガしやすい（負）
- スロースターター/早熟の体質（成長タイプの現れ方に作用）
  ※「疲れにくい（Tireless）」は選手疲労の廃止（2026-07-17）に伴い削除

## 3. 取得・喪失

- **生成時**: 才能・性格に応じて0〜数個を保有
- **後天取得**:
  - 練習の積み重ねで習得判定（長期の投げ込みで挙動が安定し「クセ球」発現、等）
  - イベントで習得（設計書03 §5.5/5.6と連動。覚醒時に有益スキル付与など）
  - 実戦経験で条件付き習得
- **喪失・克服**:
  - 負スキルはイベント/成長で克服できる余地を残す
  - 恒久喪失は原則作らない（理不尽回避）。スランプでの一時機能停止はあり得る

## 4. 元祖リスペクトの目玉スキル（少数の華）

- **「持っていたら試合が変わる」級の稀少スキルを数種**だけ用意（ばらまくと薄まる）
- 例: 「怪物」（複数分野に強力補正の超レア）、「特定投法の妙」（投法×質の職人技）など
- 出現は極めて稀。1プレイに何度も会わないからこそ記憶に残る（設計書04の怪物枠と呼応）

## 5. UI・実装

- 選手詳細（設計書06 §3.2）にスキル欄。アイコン＋短文説明（「どんな時に効くか」）
- 気づきイベント（設計書03 §5.6）でスキルの存在を部分開示（隠しスキルは育成眼で気づく）
- データ駆動: data/skills.yaml で定義（id・カテゴリ・効果・取得条件）。Claude CodeがYAML編集で追加可
- 効果は物理層/パイプラインへの補正として実装（設計書01の構造は不変）
- 検証（設計書07）: スキル保有分布と勝率寄与を集計し、壊れ性能がないか確認
