# 設計書01: 選手モデルと打席解決エンジン

方針: 元祖（高校野球シミュレーション4）に忠実＋現代化。
このドキュメントは試合エンジンの心臓部となる2システムを定義する。

---

## 0. 設計思想: 二層パラメータ構造

すべての能力値を**表示層**と**物理層**の二層で持つ。

| 層 | 例 | 用途 |
|---|---|---|
| 表示層 | ミートC、球速145、パワー62 | UI表示・育成・スカウト |
| 物理層 | 打球初速 m/s、打球角度 deg、回転数 rpm | 打席解決の実計算・3D演出 |

- 打席解決はすべて**物理層で計算**する
- 表示層→物理層の変換式を一箇所に集約（バランス調整はここだけ触る）
- 将来の3D観戦モードは、エンジンが出力する物理量をそのまま再生するだけで成立する

---

## 1. 選手モデル

### 1.1 共通属性
- 学年（1〜3年）、投打の利き（生成分布は §1.1c に従う）
- ポジション適性: 全9ポジションに適性値（元祖の「野手だが投手適性あり」を踏襲）。
  **練習・実戦で伸ばせる**（守備練習・そのポジションでの出場で上昇。守備伸びしろが効率に影響）。
  ただし**ポジション間の「近さ」で伸びの速さと上限が変わる**（隣接は易・遠隔は難）:
  内野内・外野内のコンバートは容易、内⇔外は難、**捕手と投手は特殊技能で伸びが遅く上限も低い**。
  無制限習得を防ぎ、ポジションの個性とコンバート采配（時間をかけた育成プロジェクト）を両立させる
- 精神力: 大舞台・接戦・満塁などプレッシャー状況での能力補正係数
- スタミナ（投手の試合内球数消耗＝PitchingFatigue の目安投球数。※選手の蓄積疲労は廃止 2026-07-17）
- 怪我耐性（隠し）
- **性格（4つの独立した傾向値の複合）**: 「性格タイプ名」は下記4値の組み合わせから決まるラベル。
  各値が明確な効き先を持ち、既存システムと二重計上しない:
  1. **統率傾向** → 主将にした時のチームへの影響（設計書09 §8の統率力＝統率傾向×精神力）
  2. **素直さ** → 指導の入りやすさ。素直な選手は個別指導が効きやすく伸びる／頑固は撥ね返すが自分の型を持つ
     （設計書02 §5.1の性格係数、設計書03 §3.3個別指導に作用）
  3. **勤勉さ** → 自主的な成長。勤勉は放置・オフでも伸びる／怠け者は監督の目がないとサボる（練習効率・自主練）
  4. **自己犠牲⇔目立ちたがり** → 試合での姿勢。**サイン無視はしない**が采配との相性で成功率が変わる:
     自己犠牲型は送りバント・進塁打・つなぎが得意（成功率↑、喜んで身を捧げる）／
     目立ちたがりは強振・一発狙い志向で、犠打等の指示は従うが不本意で成功率↓、チャンスの長打は上振れ
  - 一部は隠し要素として気づきイベント（設計書03 §5.6）で「？→判明」開示される
- 才能上限（隠し）: 各能力の成長キャップ
- 成長タイプ（隠し）: 早熟 / 標準 / 晩成
- **伸びしろ（隠し・分野別）**: 投手伸びしろ / 打撃伸びしろ / 守備伸びしろ の3つ。
  各分野の**成長効率倍率**（経験値の入りやすさ）。現在値とは独立 —
  入学時に優秀でも伸びしろが小さければ頭打ち、入学時ボロボロでも伸びしろが大きければ化ける。
  3年間という時間制約が実質的な到達点差を生む（天井は別に持たず、速さの差で表現）

### 1.1b 生成原則: 運動能力ベースライン（重要）
極端な専門特化（投手なのに野手能力壊滅／野手なのに投手能力ゼロ、いわゆるパワプロ型の歪み）を防ぐ。
高校球児は全員が一定の運動経験を持つ母集団であり、レギュラーなら投げれば120km/h台は出るし、
投手も打席で最低限は戦える、というのが現実。中堅校・弱小校でも同じ。

- 選手生成は2段階:
  1. **運動能力ベースライン**を先に決める（地肩・バットスピード・脚力などの素の身体能力）。
     ポジションや専門に関係なく、母集団として自然な分布から生成
  2. **専門能力**はベースラインからの**上乗せ**として生成する（投手として育つ→球速・制球が上乗せ、
     野手として育つ→打撃・守備が上乗せ）
- 帰結: 専門外の能力も身体能力相応の下限を持つ。特化は「できない」ではなく「相対的に劣る」に収まる
- **ただし変化球の球種レパートリーはベースラインに含まない**: 球速・コントロールは身体由来なので
  野手も相応に出るが、変化球は「習得・練習した球種」なので投手としての育成・経験に応じる。
  野手はデフォルトで変化球0〜1球種（緊急登板でも、ストレート＋申し訳程度の緩い球くらい）。
  → 野手が何の脈絡もなく何球種も操るのは不自然、という現実に整合させる（設計書02 §2.2の習得ルールと連動）
- **ただし経歴由来の個人差を持たせる**（重要）: 一律0〜1に固定せず、選手の来歴で変化球レパートリーに幅を出す。
  - 「中学まで投手だったが高校で野手」型 → 変化球を複数持つ野手が存在しうる
  - 「変化球は多彩で投手もそこそここなすが本職は野手」型（器用な二刀流寄り）も分布の裾に存在
  - 生成時に隠し属性「投手経歴」を確率的に付与し、それを持つ野手は変化球保有数・質が上振れする。
    大多数の野手は0〜1のまま（例外を稀少に保つことで、持っている選手が個性になる）
- 二刀流適性（投打の上乗せを両取りできる素質）は連続量として分布 → 中堅校にも「投げても打っても
  そこそこ」の選手が自然に存在し、稀に上位が二刀流の主力になる（特別枠ではなく分布の裾）

#### 1.1b-2 身体系能力は名声（才能総量）と独立に生成する
走力・地肩・スタミナといった**純粋な身体能力**は、野球の上手さ（名声＝配る才能の総量、設計書04）とは
切り離してベースライン側で独立に振る。→ 弱小校にも俊足・強肩・鉄腕が自然に生まれる。
- **走力**: 名声と独立。「弱小校の俊足1年」が出る。機動力野球（設計書11校風）や盗塁の足場になる
- **地肩（arm strength）**: 名声と独立。「弱小校の強肩捕手」「地方の大砲級の外野肩」が出る
- **スタミナ**: §1.1eの通り独立管理（目安投球数）。「弱小校の完投型」が出る
- これらは総合力に埋もれさせず、身体系だけ別枠で分布させる（能力表では他能力と並ぶが、生成ロジックが別）

#### 1.1b-3 地肩と球速・送球速度の関係（完全連動でも独立でもない）
「地肩を鍛えれば球速が上がるだけ」にはしない。地肩は土台だが、球速には投手固有の要素が乗る。
- **地肩（身体能力・共通の土台）**: 腕を速く振れる素質。野手の送球速度と投手の球速、両方の下敷き
- **野手の送球速度（肩）** ≒ ほぼ地肩で決まる（送球は投球ほど専門技術を要さない）
- **投手の球速** = 地肩 ＋ **投手センス／投球メカニクス補正**（フォーム・リリース効率・下半身の使い方）。
  投手として練習しないと球速は頭打ち → 「地肩は強いが投球素人で球速そこそこ」の野手、
  「地肩は平凡だがフォーム完成で球速が出る」技巧派投手が両立する
- 帰結: 肩を鍛えれば球速も多少は上がる（地肩が共通のため）が、投手センス／投球練習がないと伸び切らない。
  相関はあるが完全連動ではない（設計書02 §1.2の変換式に投手センス補正を噛ませる）

### 1.1c 投打の利きの生成分布（現実準拠）
「投げる手」と「打つ手」は独立ではなく、条件付きで生成する。日本野球の育成文化を反映する。

- **投げる手**: 左利き人口は約1割。ただし左投手は重宝されるため上位ほど左比率が上がる。
  - 一般母集団: 右 約88% / 左 約12%（左利き人口相当）
  - 投手として育つ層・強豪ほど左比率を上げる（プロ級で右7:左3程度に近づく＝左のプレミア）
- **打つ手**（投げる手からの条件付き）:
  - 右投げ → 右打ち 約60% / **左打ち 約40%**（右投げ左打ちは一塁到達・対右投手有利のため意図的に量産される。俊足・巧打タイプほど左打ちに寄せる）
  - 左投げ → 左打ち ほぼ100%（**左投げ右打ちは絶滅危惧種**。ごく稀にのみ生成）
- 帰結: 右投げ左打ちが多数派の一角を占め、左投げ野手は希少（守備は一塁・外野に限られる → §1.1の
  ポジション適性生成と連動）。左腕投手はレアリティが高く、対左打者の価値と噛み合う
- スイッチヒッター（両打ち）は少数（数%）。生成時に稀に付与
- 検証（設計書07）: 生成集団の投げる手・打つ手のクロス集計が上記比率に収まるか確認

### 1.2 野手能力
| 能力 | 物理層での役割 |
|---|---|
| ミート | コンタクト確率、芯を捉える精度 |
| パワー | 打球初速の上限 |
| 弾道 | 打球角度の分布（ゴロ型〜フライ型） |
| 選球眼 | ボール球スイング率、際どい球の見極め |
| 走力 | 塁間タイム、守備範囲にも寄与（純粋な足の速さ） |
| バント | 犠打・セーフティバントの成功率 |
| 盗塁 | スタートの速さ・盗塁成功率（走力とは別。判断と技術） |
| 走塁判断 | 進塁の判断力・スタートの好さ・本塁突入/自重の的確さ |
| 肩 | 送球速度（送球の速さ） |
| 送球精度 | 送球の正確さ。低いと悪送球・逸れが発生（守備解決で中継・カバーの連携を誘発） |
| 守備 | 打球判断・初動の速さ・守備範囲 |
| 捕球 | エラー率、捕手はパスボール率（元祖踏襲） |

### 1.3 投手能力
| 能力 | 物理層での役割 |
|---|---|
| 球速 | 最高球速 km/h |
| コントロール | 狙った位置からの散らばり（2次元正規分布のσ） |
| スタミナ | 球数による能力減衰カーブ |
| 球種レベル | 球種ごとの変化量・回転数 |
| 投手センス | 配球の質、投手系能力の成長効率（元祖踏襲） |

- 投法: オーバー / サイド / アンダーの3種フラグ（元祖踏襲）。**データの箱のみ保持し、球質・対左右への
  補正ロジックは現フェーズ実装しない**（全員オーバースロー相当で計算。詳細は設計書02 §2.2）
- 暴投・パスボール: コントロール×捕手の捕球で発生（元祖踏襲、劇的シーンの源泉）

## 2. 打席解決パイプライン（1球単位）

1球ごとに以下の6段階で解決する。カウント・球数の概念を持つ（現代化ポイント）。

```
① 配球決定 → ② 投球生成 → ③ 打者判断 → ④ コンタクト判定 → ⑤ 打球生成 → ⑥ 守備解決
```

### ① 配球決定
- 自チーム詳細モード: 監督（プレイヤー）の指示
- 自動時: 投手センス＋捕手リード＋カウント＋打者の弱点から球種・コースを選択

### ② 投球生成
- 狙いコースからの誤差をコントロール由来のσでサンプリング
- 球速・回転数・回転軸を決定 → 軌道パラメータ確定（3D演出と共有）

### ③ 打者判断
- コース位置×選球眼×カウントでスイング / 見逃しを判定
- 見逃し → ボール / ストライク進行。四球・三振はここから自然に発生

### ④ コンタクト判定
- ミート vs（球速＋変化量＋キレ）→ 空振り / ファウル / フェア
- タイミングのズレ量を保持（⑤で打球方向に影響）

### ⑤ 打球生成
- **打球初速**（パワー・芯の捉え）
- **打球角度**（弾道・タイミング）
- **打球方向**（引っ張り／流し、タイミングのズレ）
- この3値が打席結果の本体。ヒット性か凡打かはまだ決まっていない点が重要

### ⑥ 守備解決
- 打球の着地点・滞空時間を弾道計算（重力＋空気抵抗＋マグヌス力の数値積分）
- 各野手の初動（守備）×走力で到達可否のインターセプト判定
- 到達時: 捕球判定（エラー率）→ 送球 vs 走者の塁間タイム
- ヒット・アウト・エラー・進塁がすべて幾何計算の帰結として出る
- 各野手の役割（打球処理・ベースカバー・中継・バックアップ）は、打球タイプ×走者状況で決まる**守備陣形ルール表**（データ駆動）から割り当てる

## 3. モード両立（元祖踏襲）

| モード | 動作 |
|---|---|
| 詳細モード | 1球ごとに指示介入可。同エンジンを逐次実行 |
| 高速モード | 自動配球・自動指示で無停止実行、結果のみ表示 |
| 裏試合（他校同士） | 簡易集計モデル。詳細エンジンの統計分布に整合するようキャリブレーション |

## 4. 実装・検証方針

- 試合エンジンは **UnityEngine非依存の純C#ライブラリ** として実装
- 乱数はシード付き（同シード同結果の決定論。テスト・リプレイ・VTR機能の土台）
- ヘッドレスで1万試合シミュレート → 打率 .250〜.280、本塁打率、四球率、防御率の分布を現実の高校野球統計に寄せる
- バランス調整は「表示層→物理層の変換式」の係数のみで行う
