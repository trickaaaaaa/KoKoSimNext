# 設計書09: 試合采配システム（サイン・守備指示・伝令）

試合中の監督の介入を**3層**に整理する。
現実の野球と同じく「サインは無制限、伝令は回数制限」の構造を採る。

| 層 | 頻度 | 内容 |
|---|---|---|
| サイン | 1球ごと・無制限 | バント、盗塁などの作戦指示 |
| 方針 | 常時設定・随時変更 | 配球方針、守備陣形、走塁積極度 |
| 伝令 | **攻撃・守備 各3回/試合** | 精神面への介入（現実の高校野球ルール準拠） |

---

## 1. 攻撃サイン（1球ごと、詳細モードの作戦パネル）

| サイン | 効果 | リスク |
|---|---|---|
| 強攻 | 通常打撃 | — |
| 待て | 1球見送り（四球狙い・球数稼ぎ） | ストライクを取られる |
| 送りバント | 設計書02 §4.3の解決式 | 小フライ併殺、失敗 |
| セーフティバント | 走力依存の内野安打狙い | 同上 |
| スクイズ | 設計書02 §4.4 | ウエストで挟殺の大惨事 |
| 盗塁 | 設計書02 §4.2 | 盗塁死 |
| エンドラン | 走者スタート＋打者は必ず当てにいく（ミート補正↑・パワー補正↓、ゴロ性向↑） | 空振り→走者憤死、ライナー併殺 |
| バスター | バント構えから強攻（相手バントシフト時に有効） | 準備不足の打撃補正↓ |

- サインの成否には**相手バッテリーの読み**が絡む（捕手リード＋投手センス vs 状況の意外性）。
  読まれた場合はウエスト・ピッチアウトで対抗される

## 2. 守備指示

### 2.1 守備陣形（元祖のボタン構成を踏襲）
- **内野**: 前進 / 普通 / 後退（バックホーム態勢 ⇔ 併殺・長打警戒）
- **外野**: 前進 / 普通 / 後退（浅い打球ケア ⇔ 頭上を抜かれない）
- 現代化で追加: バントシフト（一・三塁手チャージ、二遊間がベースカバー回転 → 設計書01の守備陣形ルール表の初期配置を差し替え）
- 陣形は守備解決（幾何計算）の**初期守備位置そのもの**なので、効果は自然に物理から出る:
  前進守備はゴロの本塁封殺率が上がる代わりに、間を抜かれると長打になる

### 2.2 配球方針（自動配球への重み付け）
| 方針 | 配球AIへの影響 |
|---|---|
| おまかせ | 捕手リード＋投手センスの素の判断 |
| 直球中心 | ストレート比率↑（押せる投手向け） |
| 変化球中心 | 変化球比率↑（球威で劣る時） |
| コントロール重視 | ゾーン内比率↑・四球回避（球数も節約） |
| 低め徹底 | 低目比率↑・長打抑制、暴投リスク微増 |
| 強気の内角 | 内角比率↑・打者の踏み込み抑制、死球リスク増 |

- 詳細モードの1球指示は方針より常に優先（方針は「指示しない球」に効く）
- ウエスト / 申告敬遠は独立コマンド（元祖踏襲）

## 3. 伝令（攻撃・守備 各3回/試合）

- 回数: **攻守それぞれ1試合3回まで**。延長では1イニングにつき1回追加（現実ルール準拠）
- 効果はすべて**精神系**（設計書02 §3のプレッシャー指数まわりに作用）:

| 種類 | 効果 |
|---|---|
| 守備伝令（マウンドへ） | 投手の「動揺」状態を解除。プレッシャー指数の負補正を数打席の間緩和。連打時の立て直し |
| 攻撃伝令（打者・走者へ） | 打者の緊張緩和（精神力の負補正を一時中和）。走者への確認（暴走系の判断ミス率↓） |

- 効果量は監督の采配能力と選手の性格で変動
- 「ここで使うか、9回に取っておくか」という資源管理が采配の華になる設計

## 4. 高速モードとの関係

- 方針（配球・陣形・走塁積極度）は高速モードでも常時有効
- サインはAI代行（監督の采配能力で質が変わる）
- 伝令は高速モード中でも任意のタイミングで一時停止して使用可能

> **観戦ライブの停止粒度はこの高速モード像に一本化した（issue #173・2026-07-24）。**
> 試合ライブ観戦は「打席ごとに手動で進める」を廃し、終局まで**連続自動進行**する。プレイヤーは
> **任意のタイミングで「タイム」をかけて中断**し、采配一式（交代・サイン方針・伝令効果）を差し込んで再開する。
> 「タイム」は**有限資源＝伝令**（§3・攻守各3回・延長で毎回+1）を消費する。攻守どちらの伝令かは
> その場面の表裏×自校の先後（攻撃中＝攻撃伝令／守備中＝守備伝令）で自動判定し、残数を常時表示、残0では不可。
> これは design-15 §0 の「停止粒度＝打席頭のみ＋手動一時停止」を上書きする決定である。

## 5. エンジン接続（実装ポイント）

| 采配 | 接続先 |
|---|---|
| 攻撃サイン | パイプライン③打者判断・②の前の走者スタート判定 |
| 守備陣形 | 守備陣形ルール表の初期配置パラメータ（設計書01 §2⑥） |
| 配球方針 | パイプライン①配球決定の重みベクトル |
| 伝令 | プレッシャー指数の補正項＋状態異常（動揺）の解除 |

- UIは設計書06 §3.4の作戦パネルに実装。伝令は残回数ランプ（3灯）で常時表示

## 6. 指名打者（DH）

- ラインナップ構成の切替として実装。打席解決パイプライン（設計書01 §2）は変更しない
- DH有効時: 打順9人のうち1枠が**守備位置を持たない打撃専門（DH）**、投手は打順外
- DH解除の分岐（現実ルール準拠、監督采配として発生）:
  - DHの選手を守備に就かせる → 以降そのゲームはDH消滅（投手が打席へ）
  - 投手をそのまま打席に立たせる交代 等
- 育成・スカウトへの波及: 「守備は難があるが打力が図抜けた選手」の居場所が生まれる。
  弾道・パワー特化型の価値が上がり、起用の幅が広がる
- 設定トグル（設計書05 §1.3）。年代連動で導入年以降は自動ON

## 7. タイブレーク

- 延長は**無死一・二塁＋前イニングからの継続打順**で開始（現実の高校野球ルール準拠）
- 開始イニングはトグルで設定可能（規定回から / 延長即時 など、年度で変わるためデータ化）
- エンジンは走者配置と打順を初期化してイニングを回すだけ。既存の試合エンジンで完結
- 采配への影響: タイブレークでは送りバント・スクイズ（設計書09 §1）の価値が跳ね上がる。
  伝令の残回数（§3）をここまで温存したかどうかが勝負を分ける設計
- 設定トグル（設計書05 §1.3）。年代連動

## 8. 主将（キャプテン）

チームに1名指名。**実力ではなく統率力（性格・精神力ベース）で選ぶ**のが本来の姿で、
「実力は劣るが精神的支柱の主将を、無理にスタメンで使うか？」という采配ジレンマを核にする。

### 8.1 効果
- **試合中（在場時）**: 主将がグラウンドに立っている間、チーム全体の**プレッシャー指数の負補正を緩和**
  （設計書02 §3。緩和量は主将の統率力＝性格×精神力に比例）。ベンチにいる間は効果が大きく減衰
- **チーム運営**: 士気・結束にボーナス。設計書03の選択肢イベント（不和・サボり等）の好転率が上がる
- **育成**: 主将本人の精神力・実戦系能力の成長がやや促進（重責が人を育てる）

### 8.2 起用ジレンマ（設計の主眼）
- スタメン起用: 打線の総合力は下がるが、試合を通して全体が締まる（終盤の失点・凡走を抑える）
- ベンチ/代打起用: 打線は最強で組めるが、接戦終盤にチームが浮き足立ちやすい
- 統率力の高い主将ほどこの天秤が重くなる → 「無理にでも使うか」の悩みが毎試合発生
- 主将が不在（ベンチ外・故障）だと運営・試合の両ボーナスを失う → 主将の代えの育成も課題に

### 8.3 サイクル
- 3年生引退→新チームで主将を選び直す（設計書03の年間カレンダー連動）。指名は監督采配
- エンジン接続: プレッシャー指数の補正項に「主将在場フラグ×統率力」を加算
