## 1. 能力値スケール

### 1.1 基本
- 内部値: **1〜100の整数**（物理層への変換はこの値から行う）
- 表示: **数値＋等級の併記**（例: ミート 68 [C]）

| 等級 | 内部値 | 高校野球での位置づけ |
|---|---|---|
| S | 90〜100 | 世代の怪物（数年に一人） |
| A | 80〜89 | ドラフト上位級 |
| B | 70〜79 | 強豪校の主力 |
| C | 60〜69 | 強豪校レギュラー級 |
| D | 50〜59 | 一般校の主力 |
| E | 40〜49 | 平均的な部員 |
| F | 30〜39 | 発展途上 |
| G | 1〜29 | 初心者・素材型 |

- 球速のみ等級ではなく **km/h直接表示**（レンジ110〜165、ただし分布は下記§1.1bに従う）
- **スタミナも等級ではなく「目安投球数」で直接表示**（例: スタミナ 110球）。他能力と独立に生成し、
  他の投手能力（球速・制球等）のランクに縛られない → 「弱小校でもスタミナだけはある完投型」
  「強豪の細い線のエース（球は一級だが長いイニングは持たない）」が自然に生まれる（詳細は §1.1e）

### 1.1b 球速の分布モデル（現実準拠・2025年基準／最速km/h）
表示・生成する球速は各投手の**最速**。「160km/hがバンバン」は防ぎつつ、
甲子園級のエースは最速140台が普通、という実態に合わせる。
（注: 甲子園の「直球平均134.8km/h」という統計は全投球の平均球速であり、最速や投手個人の球速感とは別物。混同しないこと）

- **母集団は二層（最速ベース）**:
  - 全4000校の一般母集団: 中央値**120km/h台**。130km/h出れば地区で注目、というのが大多数の実像
  - 甲子園出場級（頂点）: **エースは最速140台がボリュームゾーン、二番手でも140前後**
- **節目の意味**（最速での肌感）:
  - 140km/h前後 = 甲子園出場校の主力ライン（近年はエース級ならほぼ到達）
  - 145km/h超 = 評価がはっきり上がる。プロ注目クラス
  - 150km/h超 = 世代トップclass（甲子園で150超は歴代でも20人規模だが、近年増加傾向）
  - **155km/h = 甲子園の歴史的最速級**（公式最速155で3投手が並ぶ）
  - **160km/h以上 = 原則出さない。出るなら数年〜十数年に一人の「事件」**
- 実装: 正規分布ベースで、一般母集団と強豪校で平均をずらす（強豪校ほど高球速投手が集まる）。
  上限付近（150超）は右裾を絞り、160+は超レア才能＋好条件が揃った時のみ
- 経年インフレ（トレーニング進化）を表現するなら基準年から年次で平均を微増させる係数も可

#### 1.1d 試合中の球速は毎球ばらつく（最速は上限、常時はそれ以下）
選手が持つ球速値（例: 最速141km/h）は**カタログ上限**。試合では毎球MAXが出るわけではなく、
**最速を天井とした分布から1球ごとにサンプリング**する。常時球速は最速より数km/h下、たまに最速付近。
- **基本のばらつき**: 全力投球でも1球ごとに±の揺れ。常時は最速−3〜5km/h程度、力を込めた球で最速付近
- **スタミナで低下**: 球数を重ねると終盤に球速が落ちる（設計書02のスタミナ減衰カーブ）
- **調子・試合限りの好不調で上下**: 好調な日は走る、悪い日は本来より数km/h出ない（§3.3 / §3.3b）
- **場面で最速が出る**: ここ一番で力んだ1球だけ最速が出る（精神力×プレッシャー §3.1）
- 帰結: 「終盤に球速が落ちてきたエースを見て継投判断」「今日は球が走っていないと察する」
  「9回絶体絶命で普段出ない最速」が全てこの球速分布の上で自然に表現される
- 実装: 打席解決で投球ごとに球速をサンプリング（最速を上限に、スタミナ/調子/場面の補正を乗せる）。
  表示（選手画面）は最速を出し、試合中の実況・データでは実測球速を出す
- 検証（設計書07）: 全投手＆「甲子園級」部分集合それぞれで球速ヒストグラムを出力し、
  甲子園級の中央値が140台に来るか・150超の希少性・160超の事件性を確認

#### 1.1e スタミナ（目安投球数＋消耗カーブ・他能力と独立管理）
スタミナは他の投手能力ランクと**別管理**。表示は**目安投球数**（例: 110球）、内部は消耗カーブを持つ。
- **表示 = 目安投球数**: 「本来の力を保てる球数」の目安。80球で垂れる投手、130球投げても落ちない鉄腕、が一目でわかる
- **内部 = 消耗カーブ**: 1球ごとにスタミナを消費し、残量が減るほど球威（球速分布の天井）・制球（σ悪化）が低下。
  §1.1dの球速ばらつき・終盤低下はこのカーブが駆動する
- **他能力と独立に生成**: 球速・制球のランクと切り離してスタミナだけ振る。
  → 「弱小校の完投型（球は平凡だがバテない）」「強豪の細いエース（球は一級だが5〜6回で限界）」が生成される
- 球数制限ルール（設計書05）・継投判断とダイレクトに連動
- 消耗速度に効く要素: 1球の力み（下記「飛ばしていけ」）、暑さ等の環境（※選手の蓄積疲労・体質スキル「疲れにくい」は廃止 2026-07-17）

#### 1.1f 采配「飛ばしていけ」（スタミナと球威のトレードオフ）
監督の投手指示に**「飛ばしていけ / 流していけ」**のギア設定を加える（設計書09の配球方針の一種）。
- **飛ばす**: スタミナ消費を増やす代わりに、球速分布の天井・球威を一時的に上げる（力を込めて投げる）
  → 「格上相手に序盤から飛ばして先制」「ピンチだけギアを上げる」。ただし早くバテる（継投前提の采配）
- **流す**: 省エネで投げてスタミナを温存（球威は控えめ）。「大差でエース温存」「先を見据えて力を抜く」
- 場面指定でも使える（このイニングだけ飛ばす等）。伝令・配球方針と並ぶ投手起用のレバー
- エンジン接続: ギア係数を球速サンプリング（§1.1d）とスタミナ消耗カーブ（§1.1e）に乗せるだけ。
  精神力×プレッシャーの「場面で最速」とは別で、こちらは監督が能動的に選ぶ

### 1.2 表示層→物理層 変換式（初期値）
| 能力 | 変換式 | 例（値50 / 値100） |
|---|---|---|
| パワー → 打球初速上限 | 125 + P×0.82 km/h | 166 / 207 |
| 走力 → スプリント速度 | 6.0 + R×0.025 m/s | 7.25 / 8.5 |
| コントロール → 投球散布σ | 30 − C×0.22 cm | 19 / 8 |
| 地肩 → 送球速度（肩） | 90 + Arm×0.5 km/h | 115 / 140 |
| 送球精度 → 送球散布σ | 40 − Ac×0.36 cm | 22 / 4 |
| 守備 → 打球反応遅延 | 0.60 − D×0.004 s | 0.40 / 0.20 |

- **初速式の改訂（2026-07-20・Q1クローズ）**: 旧式 `100 + P×0.6`（P50→130km/h）では物理層の弾道積分で
  外野フェンス（95〜118m）をほぼ越えられず、HR率帯（2〜4%）に構造的に届かないことがPhase 1実測で判明。
  `125 + P×0.82` へ正式改訂（`data/coefficients.yaml` の `batting.exit_velo_intercept_kmh` /
  `exit_velo_per_power` 駆動）。この値で AVG≈.265 / K%≈19.6 / BB%≈8.6 / HR%≈3.1 の全帯内を達成済み。
  なお現実の高校生の打球初速最大値（金属バット・約160〜170km/h）より意図的に高めだが、これは本作の
  弾道モデル＋守備解決の総体で帯を満たすための**ゲーム内スケール**であり、実測値との一致は要件としない

- **地肩と球速・送球の関係（設計書01 §1.1b-3）**: 地肩（arm strength）は身体能力の土台。
  - **野手の送球速度（肩）** ≒ 地肩でほぼ決まる（上表）
  - **投手の球速** = 地肩を土台に**投手センス（投球メカニクス）補正を乗せて**決まる。
    概念式: 最速 ≒ base(地肩) + 投手センス×係数。→ 地肩が同じでも投手として鍛えた選手は球速が上振れ、
    鍛えていない野手は同じ地肩でも球速が出ない（「肩を鍛えれば球速が上がるだけ」を回避）
  - 肩を鍛えれば球速も多少上がる（地肩が共通）が、投手センス／投球練習がないと頭打ち。相関はあるが完全連動ではない

- **肩（送球速度）の校正は走塁タイムと同じ「秒」の時間軸で行う（§4.1bと対）**: 送球速度をkm/hの抽象値で
  決め打ちせず、「この距離をこの秒数で届けるか」で校正する。走者の塁間タイム（§4.1b: 一塁到達4.2秒・
  塁間3.4秒）が基準にあるので、守備側は「アウトにするには送球が何秒で届く必要があるか」を逆算でき、
  そこから肩の強さの意味が決まる。
  - 例: 三遊間深部→一塁（約40m）で、走者の一塁到達に間に合うか＝肩の強さの体感
  - 捕手の二塁送球（盗塁刺し）: 走者の塁間3.4秒 − 投手クイック − 捕球 から、必要なポップタイム＋送球速度を逆算
  - → 「肩A=140km/h」でなく「この肩なら深い当たりから間に合う/間に合わない」で強さが実感できる。
    守備と走塁が同じ時間軸で噛み合う（係数は設計書07の検証で合わせる）

## 2. 球種ラインナップ

### 2.1 全9球種
各球種は物理層では「球速比・回転数・回転軸」で定義し、変化量は軌道積分から導出する。

| 系統 | 球種 | 球速比 | 変化の性質 |
|---|---|---|---|
| 速球 | ストレート | 1.00 | バックスピンによるホップ成分（質はランクG〜S、§2.2） |
| 速球 | ツーシーム | 0.98 | 微シュート＋微沈 |
| 速球 | カットボール | 0.96 | 小さく横滑り |
| 曲がり | スライダー | 0.90 | 横〜斜めの大きな変化 |
| 曲がり | カーブ | 0.80 | 縦の大変化＋緩急 |
| 落ち | フォーク | 0.92 | 低回転の縦落ち |
| 落ち | チェンジアップ | 0.85 | 沈み＋緩急 |
| 内角 | シュート | 0.95 | 打者の内側へ食い込む |
| 内角 | シンカー | 0.90 | 内側へ沈む |

### 2.2 習得ルール・球種ランク
- 全投手: ストレート必修。**変化球の習得数に上限なし**（習得・維持に練習量が要るため、実質的な制約は自然に生まれる）
- **野手の変化球は0〜1球種に留まる**（設計書01 §1.1bと連動）: 球速・コントロールは運動能力ベースラインで
  相応に出るが、変化球は投手としての育成・経験に応じるため、野手が何球種も操るのは不自然。緊急登板・
  二刀流でもストレート＋緩い球1つ程度
- **野手の球種はデフォルト0〜1**: 変化球は投手としての育成・経験で習得するもの（運動能力ベースラインには
  含まない、設計書01 §1.1b）。野手は緊急登板・二刀流でもストレート＋緩い球1つ程度に留め、
  何球種も操る不自然を防ぐ。投手として育てれば野手出身でも球種は増やせる
- 球種ランク: **G〜S**（能力値と同じ等級体系で統一）。**ストレートも変化球も同じランク体系**を持つ
- ランクの内部は**2軸**（表示はランク1本に集約）:
  - **変化球**: 球威（回転数→変化量）＋キレ（回転効率→曲がり始めの遅さ・空振り誘発力）
  - **ストレート**: 球威（押し込む力・体感速度）＋キレ（手元での伸び＝ホップ成分・空振り誘発力）
- **球速とストレートランクの役割分担**: 球速(km/h)は生の速さ、ストレートランクはその球速をどれだけ
  活かせているかの質。→「球速150だがキレG（棒球）で打たれる」「球速140だがランクA（伸びる真っ直ぐ）で
  空振りの山」の技巧派が表現できる
- 詳細データ画面では変化量cm・回転数rpm・ホップ量を閲覧可能（Statcast風の現代化。分析好きは2軸を覗ける）
- 新球種の習得: 変化球練習の蓄積＋投手センスで習得イベント発生
- **投法（オーバー/サイド/アンダー）はデータの箱のみ残し、補正ロジックは後回し（現フェーズ実装しない）**:
  - 選手データに投法フラグは保持する（将来の拡張・表示用）が、現フェーズは**全員オーバースロー相当**で計算
  - 理由: 球種の得手不得手は球種ランク（G〜S）で、対左右は慣れ（§2.3）で既に表現できるため、
    投法補正がなくても投手の個性は十分出る。3D/モーションの作り分けコストも避けられる
  - 将来: α後に「アンダースローの打ちにくさ」を慣れシステムの応用（見慣れない投法への戸惑い）として
    足したくなったら有効化する。データの箱を残すのはそのため
  - （旧・保留中の補正案: オーバー=縦変化系強化 / サイド=スライダー・シュート強化＋対同側ボーナス /
    アンダー=シンカー強化＋ストレート浮き上がり・フォーク習得不可）

### 2.3 左右の対応と「左のプレミア」
左投手が球速・制球で劣っても抑えられる現象を、**投手の能力値ではなく打者側の対応力**で表現する
（能力水増しは二重計上になり、左が抑える理由が見えなくなるため禁止）。

- 打者は「見慣れない側の投手」に弱い。母集団は右投げ左打ちが多数（§1.1c 参照）＝左腕との対戦経験が乏しい
- コンタクト判定（設計書01 ④）に**対戦相手の左右への慣れ補正**を掛ける:
  - 打者の「対左投手経験」が乏しいほど、対左腕でコンタクト率・見極めにマイナス
  - 経験は実戦（対左腕との対戦）で蓄積 → 序盤は左腕に弱く、場数を踏むと克服（成長要素）
- 帰結: 左腕は希少ゆえに「多くの打者が慣れていない」→ スペック以上に抑える、が自動的に生まれる。
  左腕を打ち崩せるかは、打線に左腕慣れした打者や左打者への強い右打者を揃えられるかの采配になる
- スキル「対左投手○」（§4.5）は、この慣れ補正を最初から克服している打者を表す

### 2.3b 対戦・球種の「慣れ」（打席蓄積の一般則）
§2.3の左右の慣れと同じ枠組みを、**同一投手との対戦回数**と**同一球種の連投**にも一般化する。
「二巡目は打者が対応してくる」「同じ変化球を続けると狙われる」という現実を表現する。

- **対戦回数の慣れ（二巡目対応）**: 打者はその試合で同じ投手と対戦を重ねるほど、タイミング・球筋に
  慣れてコンタクトが上がる。初見の1打席目より2・3打席目が有利（打者一巡でエースが崩れる要因）
- **球種の慣れ**: 配球で同じ球種を連投すると、打者の目がその軌道に慣れて対応が上がる（配球AI＝
  設計書01パイプライン①に「同球種連投は慣れを進める」補正）。緩急・球種の散らしに意味が生まれる
- **重要（慣れを万能にしない）**: 慣れは球の質・配球で打ち消せる。
  - 球種ランク（球威×キレ）が高ければ、慣れられても押し込める（同じ球で抑え続けられる本格派）
  - コントロールで散らせば慣れは進みにくい（左の技巧派がコース・球種を散らして幻惑する継投の裏付け）
  - 継投で投手を替えれば対戦回数の慣れはリセット（先発→リリーフで打者の蓄積が切れる）
  - → 慣れが支配的になりすぎると球種ランク・球威を積む意味が薄れるため、係数は控えめ（設計書07で検証）
- 実装: 慣れは打者×投手／打者×球種の一時的な蓄積値（試合内でリセット）。§2.3の左右慣れ（恒久寄りの
  経験）とは別レイヤー。スキル「打者一巡」等はこの一般則の上での個人差として扱う

## 3. 精神力の発動条件

### 3.1 プレッシャー指数 P（状況から自動算出、0〜8）
| 要因 | 加点 |
|---|---|
| 大会段階（練習試合0 / 予選+1 / 甲子園+2 / 決勝+3） | 0〜3 |
| 8回以降 | +1 |
| 2点差以内 | +1 |
| 得点圏走者（満塁なら+2） | +1〜2 |
| 負けたら引退（3年夏） | +1 |

### 3.2 補正式
```
能力補正倍率 = 1 + (精神力 − 50) / 50 × 0.02 × P
```
- 精神力100・P8: **+16%**（大舞台で覚醒するクラッチ型）
- 精神力20・P8: **−9.6%**（あがり症）
- 適用対象: コンタクト率、コントロールσ、守備の判断遅延
- 投手スタミナが目安投球数を超過時は負側のみ増幅（終盤の失点劇の再現。投手スタミナ由来で選手疲労とは無関係）

### 3.3 調子（コンディション）システム
選手ごとに**絶不調〜絶好調**の状態を持つ。パワプロ風の丸いデフォルメの表情顔（ConditionFace、5段階）で一目表示（UI原則⑥・design-06 §3.2）。
「能力≠結果」を生む主要因の一つで、精神力（§3.1）とは別軸（精神力=場面補正、調子=期間の波）。

- **段階（例）**: 絶好調 / 好調 / 普通 / 不調 / 絶不調 の5段階（内部は連続値でもよい）
- **変動は数日〜数週間続く波**として持つ:
  - ランダムな波＋試合での結果・イベント（§5.5スランプ/好調）で上下
  - 毎試合ランダムに振り直さない → 「最近好調が続く選手を今のうちにスタメンで」という起用判断が生まれる
- **効果**: 打者はコンタクト率・打球初速、投手はコントロールσ・球威に補正（打席解決の係数。設計書01のパイプラインは不変）
- **可視性（相手は監督の目で見え方が変わる）**:
  - 自チーム: 常に正確に可視
  - 相手チーム: **監督の育成眼/采配（設計書04）が高いほど正確に見える**。低いと曖昧/誤認あり
    → 「相手エースが絶好調と見抜いて敬遠策」等、相手の調子を読むのも采配になる
- ムラっけスキル（設計書10）は調子の振れ幅を増幅する体質として接続
- 検証（設計書07）: 調子分布と、調子が勝敗に与える寄与が過大/過小でないか確認
- **初期化（issue #50・2026-07-21決定）**: 選手生成時（シーズン開始時ロスター・新入生とも）に、
  週次AR(1)（persistence/sigma）の定常分布 N(0, σ_stationary²) から初期値を抽選する。
  σ_stationary = WeeklySigma / √(1 − WeeklyPersistence²)（現行係数 persistence=0.75, sigma=0.28 で ≈0.42）。
  全員「普通」固定では出せない、シーズン序盤・新チーム発足直後からの個体差を再現する
  （「初週から調子で選手を選ぶ」プレイが成立する）。乱数は専用Forkストリームで抽選し、
  既存の能力ロール列とは独立（不変条件#2、決定論・分布テスト非破壊）

### 3.3b 試合限りの好不調（当日の出来）
数日〜数週間続く「調子」（§3.3）とは**別レイヤー**の、その試合1試合限りの微補正。
「ブルペンでは良かったのにマウンドでは走らない」という当日の出来を表現する。

- 試合開始時（登板時）に各選手へ**その試合限りの小さな揺らぎ**（±）を付与
  - 投手: コントロールσ・球種のキレ・球威に微補正 → 「今日はどうも制球が定まらない」等
  - 野手: コンタクト・打球初速に微補正
- **事前に見えない（試合の中で察するしかない）**: 調子アイコンが普通でも、投げてみて初めて
  「今日は走ってない」と分かる。監督は数打者の内容から読み取り、早めの継投を判断する
  → 「アイコンは平常なのに、なんか今日はおかしい。早めに動くか」という継投の妙が生まれる
- 幅は控えめ（調子§3.3ほど大きくしない）。だが接戦では継投判断を左右する
- **頻度が最重要（本末転倒の回避）**: 当日の揺らぎは**大半の試合ではごく小さく体感できないレベル**（平常運転）。
  「明確におかしい日」は**たまーに**（目安: 十数試合に一度程度）しか起きないよう、分布の裾を薄くする。
  → 普段は実力どおりに進み、シーズンに数回だけ「なんだ今日は」の事件が混じる。
  能力の積み上げが無意味になる運ゲー化を防ぎ、「基本は実力どおり＋たまに番狂わせ」の原則を守る
- 調子§3.3（見える波）と3.3b（見えない当日の出来）の二層で、「登板前に備える不調」と
  「投げて初めて分かる不調」が両立する
- ※UIでは明示表示しない。実況・成績の推移から間接的に察させる（監督の観察力が問われる）
- 検証（設計書07）: 当日揺らぎの分布が「大半は微小・大崩れは十数試合に一度」に収まるか確認。
  頻発していないか（能力より運が支配していないか）を必ずチェック

## 4. 走塁・盗塁・バント・スクイズの解決式

すべて設計書01の⑥と同じ「時間の勝負」として幾何計算で解決する。

### 4.1 走塁（進塁判断）
```
走者所要時間 = 残り塁間距離 ÷ スプリント速度(走力) + スタート遅延(走塁判断)
守備所要時間 = 打球滞空/到達時間 + 捕球 + 送球距離 ÷ 送球速度
margin = 守備所要時間 − 走者所要時間
進塁成功確率 = logistic(margin / 0.15)
```
- スプリント速度は走力、スタート遅延と「回すか自重か」の的確さは**走塁判断**で決まる（足の速さと判断は別能力）
- 進塁を試みるかは margin と監督采配（ガンガン回す / 慎重）の閾値で決定

#### 4.1b 校正目標（現実タイムに合わせ込む）
走力→速度の変換係数は、**本塁→一塁の駆け抜けタイム**を現実値に校正する（設計書07で検証）。
「距離÷速度」だけでなく打った後の走り出しも含めた実戦タイムに合わせる。
- 目安（打者走者・塁間27.4m）: 右打者 約4.2〜4.3秒 / 左打者 約4.0〜4.2秒（一塁が近い分速い）/
  俊足 3.8〜3.9秒 / 快足の上限 3.7秒台 / 鈍足 4.5秒前後
- **左右差を反映**: 左打者は打席が一塁側に近く到達が速い（内野安打が出やすい）。§1.1cの左右分布と連動
- 盗塁絡みの塁間タイム（一塁→二塁）も現実値に校正（速い選手で約3.4秒）。§4.2のクイック/ポップタイムと整合
- これらが合っていれば内野安打の頻度・盗塁成功率が自然にリアルになる（係数を机上で決めず検証で合わせる）

### 4.2 盗塁
```
走者側: リード後残り約23.5m ÷ スプリント速度(走力) + 反応遅延(盗塁パラメータ)
守備側: 投手クイック1.15〜1.45s + 捕手ポップタイム1.85〜2.25s + タッチ0.1s
```
- 走者の反応遅延（スタートの速さ）は**盗塁パラメータ**、到達の速さは走力。両者が独立して効く
- クイックは投法・けん制能力で変動、ポップタイムは肩＋捕球から算出
- margin判定は走塁と同式。けん制死のリスクをリード幅と連動

### 4.3 バント
```
基礎成功率 = 40% + バント×0.5% − 球速補正 − コース補正
```
- **バントパラメータ**が主要因（ミートは僅かに寄与）。結果分布: 犠打成功 / 内野安打チャンス / 小フライ / ファウル / 空振り
- セーフティは走力で内野安打率が上振れ

### 4.4 スクイズ
- バント成功率 × 三塁走者スタート（走塁判断） × **ウエスト（外し）判定**
- ウエスト判定: 相手バッテリー（捕手リード＋投手センス）が状況・カウントから察知
- 読まれた場合は外球 → 三本間挟殺の大惨事。ハイリスク・ハイリターン設計

### 4.5 スキル（特殊能力）枠
元祖の「スキル」概念を踏襲し、軽量な特性システムとして実装。
**走塁系（バント・盗塁・走塁）は連続パラメータに一本化したためスキルには置かない**。
スキルは数値化しにくい特性に限定。例: 逆境◎、対左投手○、ムードメーカー、故障しにくい、ビビリ（負スキル）
- **詳細は設計書10（特殊能力詳細設計）**: 有無フラグ制・後天取得あり・カテゴリ別ラインナップ

## 5. 育成システムとの接続

### 5.1 経験値方式
```
Δexp = 練習基礎値 × 施設係数 × (1 + 指導力×0.005) × 性格係数(0.8〜1.2)
        × 成長段階係数 × 伸びしろ係数
必要exp(v → v+1) = 100 × 1.05^v
```
（※疲労係数・練習強度倍率は廃止 2026-07-17。§3.2参照）
- **伸びしろ係数**: 鍛える能力が属する分野の伸びしろ（投手/打撃/守備、設計書01 §1.1）を
  成長効率倍率として掛ける。例: 投げ込みや変化球練習には投手伸びしろ、打撃練習には打撃伸びしろ。
  現在値と独立なので「入学時優秀でも伸びない／入学時低くても化ける」が生まれる。
  高い伸びしろ×早い成長段階が揃った時期が、その選手の伸び盛り
- 練習メニューは週単位で割当（打撃 / 守備 / 走塁 / 投げ込み / 変化球 / 筋力 / 休養）
- 天井は別に持たず、伸びしろ（速さ）×3年間の時間制約が実質的な到達点差を生む

### 5.2 成長段階係数（成長タイプ×時期）
3年生は夏の敗退で引退し実質4ヶ月弱しかないため、年単位ではなく**半期単位**で定義する。

| タイプ | 1年前半 | 1年後半 | 2年前半 | 2年後半 | 3年(4〜7月) |
|---|---|---|---|---|---|
| 早熟 | ×1.4 | ×1.2 | ×1.0 | ×0.8 | ×0.6 |
| 標準 | ×1.0 | ×1.0 | ×1.1 | ×1.1 | ×1.0 |
| 晩成 | ×0.6 | ×0.8 | ×1.1 | ×1.5 | ×1.8 |

- 在籍月数で加重した生涯獲得量: **早熟28.8 / 標準29.2 / 晩成31.2**（ほぼ等価、晩成がわずかに上）
- **解釈の確定（2026-07-20・Q4クローズ）**: この生涯獲得量は**全能力合計の成長予算**（分散して鍛えると
  各能力は少しずつ、絞れば単一能力を大きく伸ばせる）。単一能力あたりの獲得レベル数ではない。
  balanced自動練習で野手中核6能力の合計成長が≈27〜30（1能力平均+4〜5）に収まるよう
  `training.base_main_exp`（coefficients.yaml）で校正済み。
  なお弾道(LaunchTendency)・選球眼(Discipline)は練習メニューを持たない**型属性**として据え置き、
  育成曲線の中核平均から除外する（練習経路を将来与えるかは別途判断）
- さらに晩成は才能上限（隠し）の生成に上方補正 → 「3年夏に化ける」ドラマをシステムで担保
- トレードオフ: 早熟は1年夏から即戦力になれる。晩成は序盤に出場機会を得られないと、実戦経験でしか伸びない能力（精神力・走塁判断）が育たないリスクを負う

### 5.3 実戦による成長（練習主体を保ちつつ実戦を補完）
成長の主軸は練習（§5.1）。そのうえで実戦には2つの成長寄与を持たせる。

**(a) 実戦でのみ伸びる能力** →【実装済み 2026-07-20（Q8）: `MatchGrowthModel`。出場1試合ごとに
match_mental/lead/baserunning_exp×成長段階係数を付与、必要expは§5.1と同曲線、隠しcap（Late上振れ・
リードcapは野球脳相関）。初期分布は「未熟（中心46）→3年で開花」へ引き下げ。実戦機会は現状大会のみ】
- 精神力・走塁判断・捕手リードは**公式戦/練習試合の出場でのみ成長**（練習では伸ばせない）
- 「練習試合で経験を積ませる」元祖の醍醐味をシステムとして担保

**(b) 成功体験ボーナス（少量）**
- 試合での成功が、該当能力に**少量のボーナス経験値**を与える:
  - 好打（安打・長打・好機での一打）→ ミート/パワー等に少量
  - 好投（抑えた・奪三振・完投）→ コントロール/球種/スタミナ等に少量
  - 好守（好プレー）→ 守備/捕球/送球精度に少量
- **大舞台ほどボーナスが大きい**（甲子園での成功が飛躍に繋がる。設計書03 §5.5の「大舞台での化け」と接続）
- 量は練習成長を上回らない範囲に抑える → 「試合に出続けるレギュラーは総合的にも伸びる」不自然さを
  解消しつつ、「とにかく試合に出すのが最適解」にはしない（練習の計画性を主軸に保つ）
- 出場機会の価値が上がる → 背番号・スタメン起用（設計書06 §3.3b）が育成の意味も帯びる
