## 2. 不変条件（違反不可。CLAUDE.md より）

1. **二層構造**: 打席・打球・守備は必ず物理層（初速・角度・回転・時間）で計算。表示能力を直接確率にしない。変換式は `data/coefficients.yaml` に集約。
2. **決定論**: 乱数は必ず `IRandomSource` 注入。`System.Random` 直 new / Guid由来禁止。同シード同結果を常に保証。
3. **エンジン純度**: `engine/KokoSim.Engine/` に `UnityEngine` 参照を持ち込まない。時刻・IO・乱数は注入。
4. **データ駆動**: バランス係数・球種・イベント・校名・球場・大会形式は `data/` のYAML。調整でC#を書き換えない。
5. **統計回帰**: バランス影響変更後は Heavy を回し `data/balance-targets.yaml` の帯を満たす。

**設計方針の核**（CHANGELOG）:
- 打席解決パイプライン構造は不変。調子・怪我・スキル・プレッシャー・采配はすべて**補正係数**として物理層変換の前段に差し込む。
- **二重計上禁止**: 走塁/肩/送球精度/左右相性/チャンス逆境 は能力値・PressureModelが担当し、スキルにしない。
- **プレイヤーとAIは同じ采配システム**（`ITacticsBrain`）。AI専用の裏ロジックを作らない。

## 3. アーキテクチャの要点（次の担当が踏み外しやすい所）

### 決定論の Fork パターン（超重要）
`rng.Fork(streamId)` は親の内部状態 `_s0` を**読むだけで進めない** → 独立ストリームを作り**親の乱数列を乱さない**。新機能を追加して既存テスト・統計帯を壊さないための主力技法。使用中の magic constant:
- 氏名 `NameStreamId`、精神系 `0x3E37`、スキル `0x5C11`、校風/采配能力 `0x57A1`
- Season週次: 調子 `0xF0F0` / 怪我 `0x1213` / 成長 `0x6707`、監督成長 `0xA6A6`

### 「既定オフ＝帯不変」設計
新サブシステムは**既定で無効**＝既存挙動と1ビットも変わらないようにしてある。これが統計帯を守る鍵:
- 精神力 **Mental=50** で PressureModel は恒等
- **Tactics=null**（采配なし）で GameEngine は分岐を素通り
- **SkillSet.Empty** で SkillModel は何も作用しない
- **CaptureTimelines=false**（既定）でタイムライン捕捉ゼロコスト
- 現代ルール（DH/タイブレーク）既定オフ
新機能を足すときは必ず「未使用時は恒等」を保つこと。

### 係数の流れ
`data/coefficients.yaml` → `KokoSim.Config/CoefficientsFile.cs`（DTO, snake_case = UnderscoredNamingConvention）→ `CoefficientsBundle` → 各エンジン record。
Bundle メンバー: Aerodynamics/Mound/Pitching/Batting/Fielding/Baserunning/Fatigue/Form/**Skills/Pressure/Tactics/EnemyAi**/Injury/Growth/Insight/ManagerGrowth/Training/Roster/Nation/Career。
別ファイルの独立ローダ: `FormationsLoader`(defensive-formations.yaml)/`SkillsCatalogLoader`(skills.yaml)/`PrefFormatLoader`・`RegionalFormatLoader`(pref-formats/)。

### リポジトリ構成
```
engine/KokoSim.Engine/        # 純ドメイン（Players/Match/Season/Nation/Career/Events）
  Match/AtBat, Match/Batting, Match/Pitching, Match/Fielding, Match/Field, Match/Game,
  Match/Timeline, Match/Tactics（采配・敵AI）, Nation/Tournaments（大会形式）
engine/KokoSim.Engine.Tests/  # 単体＋統計回帰（Heavy）

engine/KokoSim.Config/        # YAMLローダ（唯一IOを持つ層）
data/                         # YAML一式（coefficients / balance-targets / events / school-names /
                              #   player-names / defensive-formations / skills / pref-formats/*）
