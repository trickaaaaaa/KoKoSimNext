# 設計書05: 大会構造・学校データ生成

「生きた全国4000校」を、権利的に安全な架空データで、かつ処理落ちせずに回すための設計。

---

## 1. 大会構造（現実準拠、元祖の「予選のリアル化」を継承）

### 1.1 年間の公式戦
| 大会 | 形式 | 意味 |
|---|---|---|
| 春季大会（4〜5月） | 県→地区トーナメント | 夏のシード権。**プレイヤー校のみ任意参加（下記）** |
| 夏の地方大会（7月） | 49地方（北海道2・東京2）トーナメント | 甲子園出場権 |
| 夏の甲子園（8月） | 49校トーナメント・抽選 | 頂点 |
| 秋季大会（9〜10月） | **県ごとに形式が異なる**（§1.5のフォーマット定義） | センバツ選考材料 |
| 明治神宮大会（11月） | 10地区大会の王者による全国（元祖4踏襲） | 秋の日本一＋**神宮枠** |
| センバツ（3月） | 32校・1月に選考イベント | 春の甲子園 |

**春季大会の任意参加（2026-07-21決定）**: 甲子園に直結しない大会のため、プレイヤー校はスキップ可能とする。
- 大会エントリー時（春季大会開始前の週ターン）に参加確認ダイアログを表示する。文言案:
  「**春季大会に出場しますか？　※優勝しても甲子園には出場できませんが、夏の大会のシード権に直結し、選手が実戦経験で成長します。**」（最終文言はUI実装時に確定）
- 参加時の意味: 夏のシード権（§1.2）＋公式戦による実戦成長（MatchGrowthModel）
- スキップ時: 夏はノーシード扱い。該当週は通常の週ターン（練習）として進行。ペナルティは設けない
- AI校（裏試合）は従来どおり全校参加。プレイヤー校のスキップは他校のシード算出に影響しない

### 1.2 トーナメント生成
- 各県の参加校数は実際の規模感に比例（数十〜190校規模まで県差あり）
- シード: 春季大会実績から算出、ブラケット対角に分散配置
- 組み合わせ抽選はシード付き乱数（リプレイ・検証で再現可能）。抽選会はイベントとして演出

### 1.3 現代ルール（設定でON/OFF可能なトグル群）
- **指名打者（DH）**: 守備に就かない打撃専門枠が打順に入り、投手は打席に立たない。
  2025年春センバツから導入された現行ルール。エンジンではラインナップ構成の違いとして扱い、
  打席解決パイプラインは不変（設計書09 §6）
- **タイブレーク**: 延長は無死一・二塁＋継続打順から開始（設計書09 §7で詳細）
- 球数制限: 1投手1週間500球（複数投手育成の戦略的意味が生まれる）
- **年代連動**: 各ルールに「導入年」を持たせ、過去年代から始めるプレイでは導入年以降に自動適用。
  「元祖の時代の野球」で遊びたい人向けに、全トグルの手動OFFも可能（デフォルトは年代に従う）

### 1.4 裏試合の処理（パフォーマンス設計）【2026-07-21改訂: 3層→2層】

> 旧設計は「フル/中解像度/集計モデル」の3層だったが、1球単位エンジン（設計書15）完成後の実測
> （Release・采配Brain付きで4000試合＝壁時計約6.6秒・CPU合計約53秒≒1試合約13ms）により
> 「全試合フルエンジン」が現実的と判明。集計モデルを別途持つと帯キャリブレーションの保守が
> 永続的に発生するため、**計算は1本化し、層の違いは「記録の解像度」だけにする**。

- **全試合を同一のフルエンジン（1球単位）で解決する**。記録は2層:
  - **自校＋注目試合**（同県強豪・甲子園）: Playbackタイムラインまで保存（後から1球再生可能）
  - **その他全国**: ボックススコア＋スタメン/交代・継投履歴＋スコアラインのみ保存（タイムラインは破棄）
- 個人成績は全試合から大会通算へ加算 → 勝ち上がり校と当たるとき「この選手、今大会打ってる」が読める（元祖踏襲）
- 統計整合は定義上保証される（全試合が同一エンジン）。旧3層の「集計モデルをフル分布へ校正」は廃止
- 週次リザルトに「地方大会新聞」を生成（元祖踏襲）。他校の結果・怪物のうわさをダイジェスト表示
- 実装論点（ロスター実体化・保存様式・保持期間・決定論/帯影響）は OPEN-QUESTIONS **Q15** 参照

### 1.5 大会フォーマット定義（データ駆動）
- 大会は「ステージの列」としてYAMLで定義する。ステージ型は3種のみ:
  - **knockout**: トーナメント（シード規則・不戦勝処理つき）
  - **round_robin**: 総当たりリーグ（勝点制。同率は直接対決→得失点で解決）
  - **group_split**: 地区ブロック分割（各ブロック内で子ステージを実行 → 上位が次ステージへ）
- **方針: 秋季県大会は47都道府県の実形式に準拠する**（本作のこだわりポイント）
  - data/pref-formats/ に県ごとの大会定義YAMLを収録。形式は年度で微変動するため、
    **基準年を決めたスナップショット**を正とし、出典をコメントで記録、後から編集可能にする
- スキーマは実形式を書き切れるように拡張する:
  ブロック分割（地理割 / 抽選）、ブロックごとの県大会進出枠、**敗者復活戦**、代表決定戦、
  シードの持ち越し規則、参加校数の実数準拠
- 開発順: Phase 4ではスキーマ＋汎用プリセット3種で先に動かし、47県の実データ収録は
  **並行タスクとして順次差し替える**（データ駆動なのでエンジン側の手戻りはゼロ）
- 秋の全国動線: 県大会 → 10地区大会 → **明治神宮大会**（11月・神宮球場・地区王者10校）
- **神宮枠**: 神宮大会優勝校の地区はセンバツ選考枠が+1（現実ルール準拠）。センバツ選考ロジックに接続
- 形式差はゲーム的にも意味を持つ: リーグ形式の県は公式戦の試合数が多い＝新チームの
  実戦経験（精神力・走塁判断など試合でしか伸びない能力）を稼ぎやすい。**大会形式がそのまま育成環境の差になる**

## 2. 学校データ生成（架空4000校）

### 2.1 校名ジェネレータ
- パターン合成で現実らしい架空名を量産:
  - 公立系: 地名＋方角/序数/台/丘/中央/北陵…（例: 青葉東、桜丘、県立第一）
  - 私立系: 学院/学園/大付属/工業/商業/実業/国際…（例: 聖凛学院、北都大付属）
- 実在地名（市区町村）は使用可、ただし**実在校名との完全一致を避けるチェック**を通す（ベストエフォート＋エディタで自由変更可能を担保）

### 2.2 学校属性
| 属性 | 内容 |
|---|---|
| 設立 | 公立 / 私立、普通 / 商業 / 工業 |
| 伝統 | 古豪 / 中堅 / 新興（名声の初期値と減衰傾向） |
| 強さティア | G〜S。初期選手の分布、AI監督の質、施設初期値に対応 |
| 地域 | 都道府県＋**県内地区**（例: 愛知5地区、北海道10支部）。激戦区（校数多＋強豪厚い県）を再現 |

- 県内地区のリストと割当方式は pref-formats のYAMLで県ごとに定義する
- **地理固定割の県**（愛知・北海道など）: 学校が地区を固定で保持。生成校は地区ごとの規模比率で割当
- **抽選制の県**（東京の一次予選ブロックなど）: 学校は地区を持たず、大会生成時に抽選（固定枠の特例=当番校のような制度もYAMLで表現）
- 新設校作成・学校エディタでは都道府県に加えて**県内地区を選択**（抽選制の県は県のみ）
- 転任オファーには地区も表示（地区ごとの競争環境の差がキャリア選択の材料になる）

### 2.3 生きた勢力図（AI校の運営）
- AI校にも簡易版の育成・世代交代・スカウトロジックを回す
- AI校の名声も実績で変動 → **古豪の凋落、新興の台頭が自然発生**し、10年遊ぶと勢力図が変わる
- 処理はティアベースの抽象更新（全4000校をフルシムしない）

### 2.4 エディタ（元祖踏襲）
- 校名・所在地・ティアの編集、新設校の追加、既存校の削除
- 自作新設校での最弱スタート（元祖踏襲の人気プレイ）
- チームのエクスポート/インポート → 友人チームとの対戦機能の土台（元祖踏襲）

## 3. 球場データ（架空60球場、元祖4踏襲）

- 各県のメイン球場＋聖地球場を架空名で生成、エディタで変更可
- パラメータ: 両翼・中堅距離、フェンス高、風傾向（聖地の浜風的な名物風）、収容人数
- **収容人数はプレッシャー指数（設計書02 §3）と連動**、球場寸法は物理層の本塁打判定にそのまま効く
- 3D化の際はこの寸法データからスタジアムモデルを生成する前提

## 4. センバツ選考ロジック（一般選考枠＋神宮枠・成績ベース）
春のセンバツは選手権（夏）と違い**選考で選ばれる**大会。前年秋の地区大会成績を最重要資料とする。
21世紀枠のような主観選考は入れず、成績から機械的に決まる範囲に絞る（実装明快・因果が分かりやすい）。

- **一般選考枠**: 各地区に配分された枠数を、秋季地区大会（設計書 §1.5、regional-tournaments）の
  成績上位から選出。地区ごとに枠数が異なる（出場校数・地域性を反映、data で保持）
  - 同成績の比較は地区大会での勝ち上がり→内容（得失点等）で解決
  - 「地区大会ベスト4なら当確、ベスト8は当落線上」といった現実の肌感を枠数で再現
- **神宮大会枠**: 神宮優勝校の地区に一般枠＋1（§1.1で実装済み）。regional-tournaments の神宮枠に接続
- **選考発表イベント**: 1月に選考結果を発表（設計書03カレンダー連動）。当落線上の自校を煽る演出。
  「秋に結果を残せば春が見える」という因果をプレイヤーに体感させる
- 総出場枠は基準年の実数に合わせる（例年32校規模。記念大会等の増枠は年度データで表現可）
