# 設計書08: 3Dアートディレクション

コンセプト: **「野球盤の駒が動き出した」**
マットな単色フィギュア質感の抽象的な選手が、エンジンの本物の弾道データで動く。
UIテーマ（スコアボード緑×チョーク白×電光アンバー）と同一世界観で統一する。

**位置づけ**: 3Dは本線ではない。本線の詳細モードは元祖踏襲の2ビュー構成
（投打対決→守備俯瞰、設計書06 §3.4）。3Dはα完成後に「カメラ＋ボール＋簡易球場のみ・人体なし」の
1週間スパイクで継続可否を判断する増築案件（設計書07 §3）。本書はスパイクが通った場合の指針。

## 1. 抽象度の線引き

| 捨てるもの（コスト爆発・情報量小） | 捨てないもの（シルエットが運ぶゲーム情報） |
|---|---|
| 顔（のっぺらぼう） | **投法3種**のモーション差（オーバー/サイド/アンダー）※投法フラグは保持するが補正ロジックは後回し（設計書02 §2.2）。3D着手時にモーション差を付けるか判断 |
| 指・手の造形 | 左右打席・利き腕（ミラーで対応） |
| フォームの個人差 | **ユニフォーム色と背番号**（チーム識別・愛着） |
| 布・髪の揺れもの | 走者/守備の位置関係の読みやすさ |
| リアルな質感・汗・土汚れ | ボールと影（2Dで実証済みの高さ表現を3Dでも維持） |

- 原則: **すべてのアセットを同じ抽象度に揃える**。選手が駒ならスタジアムも芝もマットな様式で。
  抽象度の混在（リアルな芝×のっぺらぼう）が一番「安っぽく」見える

## 2. 選手モデル仕様

- 単一のヒューマノイドリグ1体（Mixamo互換）。顔なし・ローポリ・フラットシェーディング
- 体型スケール: 標準／がっしり（パワー型）／細身 の3段階をボーンスケールで表現（任意、後回し可）
- 質感: 単色マテリアル＋ごく弱いリムライト。輪郭線シェーダは試作して判断

| 区分 | モーション |

  モーション再生は**イベント時刻へのアニメーション頭出し**だけで同期できる（ブレンド最小限）

## 4. ユニフォーム: パラメトリック方式

**アセットは1着、バリエーションはデータ。** モデル1体＋マスクテクスチャ＋色差し替えシェーダで構成し、
学校レコードに以下のパラメータを持たせる。

| パラメータ | 選択肢 |
|---|---|
| ベース色 | 白 / アイボリー / ライトグレー（高校野球の現実に寄せて3色に制限） |
| アクセント色 | 紺・黒・えんじ・赤・緑・紫・水色・橙・金茶 など約10色 |
| パターン | 無地 / ピンストライプ / 袖切替（ラグラン） |
| 帽子 | 単色 / ツートン（つば色違い） |
| 胸マーク | 校名から自動生成（漢字 or ローマ字、書体2種）→ テクスチャに焼き込み |
| ストッキング/アンダー | アクセント色に追従 |

- 組み合わせは 3×10×3×2×… で**数百通り**。アセット制作コストは1着分のまま
- 生成規則を設計書05の学校属性と接続:
  公立→白ベース＋伝統色無地 ／ 私立→ツートンや袖切替 ／ 古豪→クラシック無地 ／ 新興→現代的配色
- 学校エディタからパラメータを直接編集可能（「うちの母校の色」を再現できる、愛される機能）

### 4.1 胸マークの実装詳細
- **縦書きは採用しない**（現実の野球ユニフォームの胸マークはほぼ横のみ。縦は幟・応援旗の文化）
- テンプレートは閉集合3種: ①漢字・横一列（毛筆系書体） ②ローマ字・アーチ（ブロック体大文字） ③胸無地＋帽子1文字
- 学校レコードに「胸マーク表示名」フィールド（漢字短縮形／ローマ字形）を持たせ、生成時に校名から機械短縮
  （例: 県立桜丘高校 →「桜丘」／SAKURAGAOKA）。**短縮アルゴリズムは完璧を目指さない**——不自然なものはエディタで直せる前提
- 実装: TextMeshPro → RenderTexture に一度だけ焼き込みキャッシュ。アーチは文字を曲線に沿わせる小スクリプト
- フォントは再配布可能な OFL ライセンスのものを使用（毛筆系・ブロック体を各1）
- 導入順: 帽子1文字 → 漢字横一列 → ローマ字アーチ。**帽子1文字だけでも学校識別は成立する**のでMVPはそこから

## 5. スタジアムとカメラ

- 球場は設計書05の寸法データ（両翼・中堅・フェンス高）から**プロシージャル生成**。
  スタンドは抽象的な段々＋観客はドット的表現（収容人数＝プレッシャー指数の視覚化）
- カメラは放送の文法で3台:
  ① バックネット（投球〜打撃） → ② 打球追尾（俯瞰寄りで守備全体が見える） → ③ ベース際（クロスプレー）
- 抽象モデルはカメラが寄らない前提なので、②を基本に引き気味で運用する

## 6. 実装順序（Phase 8 内の積み上げ）

1. カメラ＋ボール＋簡易球場（軌道再生だけで既に野球に見える）
2. 駒モデルを静止配置（守備隊形）
3. タイムライン移動（2Dモックの野手トークンの3D版）
4. モーション14種の頭出し同期
5. ユニフォームシェーダ＋学校パラメータ接続
6. カメラカット演出・スタンド・音

- 各段階が独立してリリース可能。1で止めても「3D軌道ビューア」として成立する
