# 設計書04: 監督メタ・スカウト・イベントテーブル・合宿

決定事項の反映: 転任=**強制転任あり（元祖忠実）** / 指導力=**分野別** / 新入生=**ハイブリッド**

---

## 1. 監督メタ

### 1.1 監督能力（分野別・キャリアを通じて持ち越す唯一の資産）
| 能力 | 効果 |
|---|---|
| 打撃指導 | 打撃系練習の効率係数 |
| 投手指導 | 投手系練習の効率係数、投手起用のスタミナ見積り精度 |
| 守備走塁指導 | 守備・走塁系練習の効率係数 |
| 育成眼 | 個別指導「見極め」の精度、新入生の隠しパラメータ開示精度 |
| 采配 | 自動進行時の配球・作戦の質、相手の作戦察知 |

- **成長は2階建て**（プレイヤーが意識して管理する対象にはしない）:
  1. **試合経験でゆるやかに自然増**: 練習試合・公式戦をこなすと分野別能力が少しずつ上がる
     （元祖の「練習試合で指導力が上がる」を踏襲）。格上相手・大舞台ほど得られる経験がやや大きい。
     プレイヤーが積極的に伸ばす操作は不要 ＝ 気づいたら練れている
  2. **節目イベントで大きく伸びる**（下記1.1b）: キャリアの物語の起伏として、特定能力がぐっと動く
- 設計書02の練習式の指導力項を、この分野別係数に置き換える

### 1.1b 監督成長イベント（例）
設計書03のイベントシステムに監督成長フラグを持たせ、条件成立で発生。効果はその場の能力上昇。
- **名将との練習試合**: 甲子園常連校などと対戦すると、采配 or 該当分野の指導力が上昇（教えを乞う）
- **大敗からの学び**: 大差negative の敗戦後、悔しさイベント → 弱点分野の指導力が上昇
- **指導者講習会**: オフ（冬）に参加を選ぶと指導力が上昇（費用・時間とのトレードオフ）
- **プロOBの訪問**（設計書 §3.1のイベント連動）: 教え子や縁のあるOBが来訪 → 采配・育成眼が上昇
- **甲子園経験**: 甲子園の舞台を踏むと采配が大きく上昇（頂点の空気を知る）
- **名伯楽との出会い**: 稀イベント。優秀なコーチ/部長を得て特定分野が継続ボーナス
- ※いずれも「狙って連打する」ものではなく、プレイの節目で訪れる。イベントはYAMLで追加可能（設計書04 §3）

### 1.2 信頼度と名声（二軸）
| 軸 | 対象 | 上げ方 | 効果 |
|---|---|---|---|
| 信頼度 | 校内（校長・保護者・部員） | 勝敗、怪我対応、学業配慮、選択肢イベント | 部予算、強化要求の通りやすさ、残留嘆願、**部員信頼→チーム士気の基礎値** |
| 名声 | 全国区 | 大会実績、甲子園、教え子のプロ入り | 新入生の質と数、練習試合の申込、取材 |

- **信頼度の試合への効き方（間接・薄く）**: 信頼度は試合中のプレーに直接は作用しない。
  ただし部員からの信頼が厚いほど**チーム士気の基礎値が上がり**、士気が試合の粘り（接戦・終盤の
  踏ん張り）に薄く寄与する。信頼度→士気→試合、の一段挟んだ経路。
  精神力（§設計書02 §3）・主将（設計書09 §8）・伝令（§9.3）とは二重計上しないよう、
  信頼度は「土台の底上げ」に留め、場面ごとの補正はそれぞれの担当が持つ

### 1.3 転任システム（元祖忠実: 強制転任あり）
- キャリア初期は「教員監督」。**毎年3月に転任判定**があり、拒否できない
- 実績（例: 甲子園出場 or 名声閾値到達）で**「フリー監督」資格**を獲得 → 以降は残留自由・他校オファーの選択制（元祖踏襲）
- 信頼度が非常に高い場合、転任判定を「残留嘆願」で回避できることがある（救済、確率制）
- 転任時: 監督能力と名声は持ち越し、信頼度はリセット。育てたチームとの別れがドラマになる
- ゲーム的意味: フリー化までが「修行編」。弱小校を渡り歩いて指導力を磨き、フリー化後に本拠地を定めて王朝を築く、というキャリア曲線を作る

### 1.4 資金
- 収入: 学校予算（年度頭・信頼度連動）／大会報奨（勝ち進むほど）／OB・後援会寄付（名声、プロ入りOB数）
- 支出: 備品・施設（設計書03 §4）／合宿／スカウト年契約／遠征費
- 表記: 万円

## 2. スカウト・新入生生成（ハイブリッド）

### 2.1 受動枠（毎年のメイン供給）
- 3月に入部予定者が確定、4月仮入部（能力は「？」、観察と育成眼で徐々に判明）
- 人数: 基準10人前後 ± 名声・前年実績・地域補正
- **才能生成モデル（重要・金太郎飴回避）**: 名声が決めるのは各能力値ではなく**選手1人の総合力（配る才能の総量）**。
  1. 名声から選手の総合力の期待値を決める（名声Cの学校ならC相当の総ポイント）。ここにばらつきを持たせる
  2. 総ポイントを各能力へ**ランダムな重み付けで凸凹に配分**する（均等割りにしない）。
     → 総合Cでも「ミートB・走力D・肩A・守備E」のように内訳に個性が出る
  3. 配分の偏りの強さ自体もばらつかせる → たまたま平均的に配れた「オールC型」も自然に混じる
- 才能上限・伸びしろ（設計書01）も同様に分野別で凸凹に配る → 現在値だけでなく伸び方にも個性
- まれに分布の外れ値として「怪物」（突出した総合力／一芸特化）が出現。名声で出現率がわずかに上がる
- スカウト年契約（元祖踏襲）: なし / 地元 / 広域 の3ランク。契約金は年払い
  - 効果1: 新入生の**総合力の期待値**が上振れ（内訳の凸凹は維持）
  - 効果2: 入部予定者の隠しパラメータの事前開示精度が上がる

### 2.2 能動勧誘枠（寮で解禁、少数）
- 寮を建設すると**年最大2名**の特別勧誘枠が解禁
- 秋〜冬に中学生リストから対象を選び訪問。リストの情報精度はスカウト契約ランク依存
- 成功率: 名声 × 距離（寮でカバー） × 競合校の格。強豪との争奪戦イベントが発生し得る
- 位置づけ: 「補強の主役」ではなく「どうしても欲しい1人を獲りに行く」スパイス

### 2.3 詰み防止
- ポジション配分は原則ランダムだが、捕手のみ最低保証（部に捕手適性が枯渇しない下限補正）
- 補正の強度はバランス検証で調整（硬派さを損なわない最小限に留める）

## 3. イベントテーブル（データ駆動）

### 3.1 定義フォーマット（YAML → C#へロード）
```yaml
- id: ace_hidden_injury
  trigger: { type: random, weight: 3 }
  requires: { flags: [!ace_injured_this_year], condition: "ace.recent_starts >= 4" }
  text: "エースが肘の違和感を隠しているようだ…"
  choices:
    - label: "検査に行かせる"
      effects: { player: {status: rest_3w}, trust: +5, flag: ace_injured_this_year }
    - label: "様子を見る"
      effects: { player: {injury_risk: x3_4w} }
```
- Claude Codeでの量産・追加・バランス調整をデータ編集だけで完結させるための構造

### 3.2 フラグ管理
- グローバルフラグ（キャリア通算） / 年次リセットフラグ / 選手個別フラグ の3層
- 発生制御: 同一週最大1件、同一イベントにクールダウン、重み付き抽選

### 3.3 初期ラインナップ（抜粋20件）
- 固定: 入学式、仮入部→正式入部、テスト週×4、夏合宿、冬合宿、文化祭、ドラフト会議、
  プロ入りOBの挨拶訪問（元祖踏襲）、卒業式、転任辞令（3月・判定連動）
- ランダム: 怪我、雨天週、取材、プロスカウト視察、OB寄付、感染症で部活動停止
- 選択肢: エースの違和感、練習サボり、無名1年の自主練、保護者からの起用クレーム、
  部員間の不和、他校からの引き抜き噂（フリー化後）

## 4. 合宿

| 項目 | 夏合宿 | 冬合宿 |
|---|---|---|
| 時期 | 8月（新チーム結成直後） | 12月末〜1月（鍛錬期の山場） |
| 効果 | 全体経験値×2、結束↑ | 全体経験値×2.5、結束↑↑ |
| 費用 | 30万＋1人1万（合宿所Lvで割引） | 同左 |
| リスク | 怪我判定強化 | 同左（さらに高い） |

- オプション「中日に休養日を挟む」: 効果×0.9、怪我リスク大幅減
- 合宿中は選手間イベント（結束・性格変化・不和解消）の発生率が上昇
- 合宿所の施設レベルで練習効率に補正
