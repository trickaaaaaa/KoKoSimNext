## 3. 次の実装: F1 Slice 2・3（未着手）

**F1のゴール**: 併殺(DP)を **6-4-3 の送球連鎖**（野手→二塁→一塁）でタイムライン描画する。帯リスクゼロ（結果不変・乱数不使用）。

**重要な前提知識**:
- 併殺の**走者側は既に描けている**。`BaserunningModel` のDPは `RunnerMove(r1, 1, 2, Out:true)`（二塁封殺）を出し、`TimelineBuilder.AppendRunnerLegs`（`GameEngine.cs:448` で呼ぶ）が封殺レッグに変換済み。打者の一塁アウトも既存。
- **足りないのはボールの送球連鎖だけ**。今 `BuildBattedBall` は内野ゴロで「野手→一塁」1本しか出さない。DP時は「野手→二塁→一塁」の2本にしたい。
- 既知の癖: `ApplyInPlayOut` は**フライ凡打でも**r1あれば `DoublePlayProb` を引きうる（幻フォース）。描画時は `!play.IsFly` でガードし、フライ由来は連鎖差し替えしない（Slice 2/3で対応）。

### Slice 2 — DP送球連鎖ビルダー（`TimelineBuilder` の純関数・単体テスト）
- 新メソッド `TimelineBuilder.AppendDoublePlayThrows(PlayTimeline tl, FieldingPlay play, FieldGeometry field, double batterArriveAnchor)`。
  - 既存の `Throw`（野手→一塁1本）を除去し、**野手処理点→二塁→一塁**の2本 `Throw` に差し替え。
  - ゴロの`Roll`終端(=捕球点=`play.LandingX/Z`)を1本目の始点に。二塁点/一塁点は `FieldGeometry.SecondBase/FirstBase`。
  - 時刻: 1本目(→二塁)は捕球後に到達、2本目(→一塁)は `batterArriveAnchor`（打者レッグ終端）にアンカー。送球速度は演出用固定定数（モックは30〜32 m/s）、握り替えも小さな固定値。全て表示専用=帯無影響なので実装で決めてよい。
  - `!play.IsFly` && 内野ゴロ以外は何もしない（元のtlを返す）。
- テスト（`Match/Timeline/TimelineTests.cs`、合成 `FieldingPlay`＋DP想定）:
  - ボールに`Throw`2本、順に 野手→二塁、二塁→一塁。
  - 時刻単調＆連続（Roll終端=1本目始点、1本目終端≈2本目始点、二塁到達＜一塁到達、全T1≤Duration）。
  - フライ時は差し替え無し（回帰）。

### Slice 3 — GameEngine で結線＋統合テスト
- `GameEngine.cs:448` の `AppendRunnerLegs` 後に、`runnerMoves` に封殺（`Out && FromBase==1 && ToBase==2`）があり `res.Play` が内野ゴロ(!IsFly)なら `AppendDoublePlayThrows` を適用。打者レッグ終端(=一塁到達レッグT1)をアンカーに渡す。
- テスト:
  - 複数シードで捕捉試合を回し、封殺レッグ（二塁・Label走・OutAtEnd）を持つタイムラインを最低1件見つけ、そのボールに**二塁到達→一塁到達の2連続Throw**があることを検証。
  - 非DPゴロは従来通り1本Throw（回帰）。
  - `Capture_DoesNotAffectGameOutcome`／決定論テスト緑のまま。
- **DoD**: `Category!=Heavy` 全緑。Heavyは不要（結果不変）だが念のため1回。完了後メモリ更新。
